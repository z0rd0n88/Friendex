## Findings

### LOW

- **M2 vacuous-mutation hazard** —
  `tests/adapters/discord_bot/test_error_handler.py:119` asserts
  `embed.description == error.user_facing_message`. Because
  `DomainError.__init__` calls `super().__init__(user_facing_message)`,
  `str(error)` equals `user_facing_message` for every DomainError subclass.
  A regression where `build_error_embed` (or the handler's call site) used
  `str(error)` instead of `error.user_facing_message` would pass this test
  vacuously. Concrete fix: harden one DomainError test with a subclass
  whose `user_facing_message` is intentionally different from
  `Exception.args[0]`, e.g. construct via
  `class _DivergentError(DomainError): pass; err = _DivergentError("not-the-str");
  err.user_facing_message = "actual-message"` and assert
  `embed.description == "actual-message"`. Non-blocking — the embed
  builder (`embeds.py:686-697`) is already the correct construction and is
  separately tested in Phase 10.

### INFO

- **M4 — main.py has no runtime test** — by signoff Q2 design. Phase 14
  will replace the `NotImplementedError` with real bot construction;
  whatever tests cover bot startup belong to Phase 14. Not a finding.

- **`_make_liquidation_factory` builds a fresh `TradingService` per call**
  (container.py:378-390). Two `liquidation_service_factory("g")` calls
  produce two distinct trading services. They share the same guild lock
  via the shared `LockManager`, so `_cover_internal`'s "lock-already-held"
  contract still holds across calls in practice (the lock is keyed on
  guild_id, not service identity). Flagged for Phase 14 awareness — if
  Phase 14 caches the trading service for the cogs/listeners but the
  liquidation task continues to receive a fresh one, that asymmetry is
  fine but should be documented in the digest. Captured in the digest's
  Phase-14 honour list.

- **Work-baton type-ignore miscount** — baton 001 says "3 total" but then
  enumerates only 2 (the third lives in tests). Cosmetic. Captured here
  for completeness; baton-passs are append-only so no edit needed.
