## Where things stand

Sub-unit 6c is **code-complete and gate-green** (changes not yet committed). The
SQLAlchemy-backed user repository — the richest aggregate — is implemented and
proven end-to-end against an in-memory SQLite engine with FK enforcement ON.

Added `src/friendex/adapters/persistence/user_repo.py` →
`SqlUserRepository`, constructed with an `async_sessionmaker` (matches the
persistence pattern). It implements every `IUserRepo` method — `get`, `upsert`,
`delete`, `list_all`, `list_active_in_last` — against the user-aggregate ORM
tables (`UserORM` + `LongPositionORM` + `ShortPositionORM` + `ActivityBucketORM`
+ `VoiceUniqueChannelORM`). Conformance is **structural, no inheritance**: a
typed `conforming: IUserRepo = repo` assignment in the test is what mypy gates,
and mypy passes clean.

Design choices honoring prior digests:
- **`upsert` = whole-aggregate delete-then-insert** in one transaction (`merge`
  the scalar row, delete owned children, re-insert from the domain object). Keeps
  the mapper a pure function of the aggregate; stale children never linger
  (verified by `test_upsert_replaces_existing_aggregate`).
- **`delete` does NO hand-rolled child cleanup** — it issues a single parent
  `DELETE` and relies on DB-level `ON DELETE CASCADE` (ADR-0002 / 6a digest).
- **Immutable mapping** — builds fresh domain objects from rows, never mutates.
- Money stays exact `Decimal` (value + quantisation); datetimes stay tz-aware
  UTC, both preserved across the boundary.

Also added `SqlUserRepository` re-export to
`src/friendex/adapters/persistence/__init__.py` (plan's "Files modified" item;
later sub-units extend `__all__`).
