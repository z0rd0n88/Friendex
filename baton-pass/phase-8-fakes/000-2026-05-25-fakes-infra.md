## Key implementation decisions

- Storage: `dict[(guild_id, id), Aggregate]`; SystemState keyed by `guild_id`
  alone. Immutable: store references, replace wholesale on upsert.
- Boundaries mirror the SQLAlchemy adapters exactly:
  - `list_active_in_last`: `last_activity >= now - seconds` (inclusive).
  - cooldown `get`: active iff `expires_at > now` (exclusive); `get` has the
    adapter's extra `*, now: datetime | None = None` keyword (superset of the
    Protocol, still conforms).
  - `purge_expired`: removes `expires_at <= now` (inclusive); `prune_history`:
    drops `timestamp < cutoff` (point at cutoff kept).
- `default_settings` fixture builds `Settings(discord_token="test-token",
  _env_file=None)` so it never reads a `.env`.
