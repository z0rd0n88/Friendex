## Design decisions

- `/fund create` confirmation: cog will use `build_fund_info_embed` after the
  create call by passing the returned HedgeFund + computed APYs. Cleaner than
  an inline embed and reuses the kw-only builder (Phase 10 digest convention).
- `/fund invest` per Open-Q5: do NOT catch NotImplementedError. The
  trading_service-style positional convention from 8c is mirrored by passing
  invest's args positionally to FundService.invest(...) (signature:
  investor_id, fund_id, amount).
- AllowedMentions.none() on EVERY send in fund_cog (mutations embed user-input
  fund.name; reads still apply for consistency). Trading cog mentions are from
  service results (real Discord snowflakes only) so the kwarg is still applied
  as the project convention.
- Money floats from Discord -> Decimal via `Decimal(str(amount))` exactly.

## Open questions / risks

- `app_commands.Group` subcommand registration: the FundGroup is a top-level
  `app_commands.Group(name="fund")`. The Cog class itself wraps the group
  instance and registers it on the bot.tree in Phase 13. Pattern: define
  FundGroup as a Group subclass (not instance attribute) with the
  `__init__(self, *, fund_service_factory, settings)` ctor — discord.py
  supports this and Phase 13 can `bot.tree.add_command(FundGroup(...))`.
