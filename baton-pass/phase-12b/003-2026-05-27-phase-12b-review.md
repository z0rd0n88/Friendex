### LOW (carry-forward; do not block 12b close)

1. **CF-2 mutation pin is partial — the inner re-read alone defeats the
   race even without the LockManager lock.** During M4 I first ran a
   weaker mutation that kept the `await self._ping_sessions.get(...)`
   re-read inside `_reward_for_session` but removed the `async with
   self._locks.locked(self._ping_lock_key(...))` wrapper. That mutation
   left the test GREEN — both responders correctly fell out to
   `extra_joiners`. The test only goes RED when the *entire* lock-block
   (including the inner re-read) is replaced with use of the outer
   `session` snapshot directly. Net effect: a future regression that
   removes only the LockManager guard (keeping the re-read) would slip
   through this test even though it removes the CF-2 invariant.
   **Reason it survives anyway:** in single-threaded asyncio the inner
   `VoicePingSessionStore._lock`-guarded `get` happens to serialise reads
   with prior writes via FIFO lock-queue scheduling, masking the race in
   this barrier scenario. The lock is still load-bearing for the *general*
   invariant — non-co-located readers, multi-task scheduling under
   `uvloop`, future refactors that drop the inner re-read — so this is a
   LOW (test gap), not a HIGH (logic bug).
   **Fix:** add a second test that locks the inner `_ping_sessions` store
   open during one responder's hold so the second responder cannot
   serialise via the store lock; or a smaller-surface test that swaps in
   a no-op `LockManager` and asserts behavior degrades. File:
   `tests/application/test_voice_ping_service.py`. Defer to phase-13.
   Cross-ref: phase-8a digest §LOWs.

2. **`_BarrierPingSessionStore._barrier_fired` is set after every
   post-barrier `list_all`, but on a second listener tick the barrier
   would unblock instantly** (test correctness is fine because the
   helper is one-shot, but if future tests reuse the helper across
   waves the flag's per-instance, single-fire semantics need a doc
   comment). File: `tests/application/test_voice_ping_service.py:473-485`.
   LOW, doc only.
