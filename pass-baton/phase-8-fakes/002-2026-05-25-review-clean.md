## Findings

No CRITICAL / HIGH / MEDIUM.

- **LOW ‚Äî `get` returns the stored mutable reference (divergence from adapters).**
  `tests/application/fakes/fake_repos.py` (all `get`/`list_*` methods, e.g. :60,
  :108, :171). Domain dataclasses are plain `@dataclass` (mutable), and the
  fakes hand back the *same* stored object. The real SQLAlchemy repos rebuild a
  fresh domain object on every read, so a caller that mutates a returned
  aggregate in place sees the change on the next fake `get` but NOT on the real
  repo (verified: fake returns mutated `cash_balance=999` where SQLite would
  show 100). Only bites code that violates the project's immutability discipline
  (never mutate; replace wholesale via `upsert`). Fix if desired: `copy.replace`
  / `dataclasses.replace` on store and on return, or document the invariant.
  Acceptable to defer ‚Äî 8a‚Äì8f must follow immutable style regardless.
- **INFO ‚Äî AC1 signature test is one-directional.**
  `test_fake_method_signatures_match_protocol_signatures`
  (`test_fake_repos.py:157`) only asserts the fake does not *drop* Protocol
  params (allows extras, e.g. cooldown `now`). The real conformance guarantee is
  mypy structural assignability (`repo: IXxxRepo = FakeXxx()`), which the gate
  enforces. Sound as-is; noted so a future reader doesn't over-trust the
  runtime check alone.
