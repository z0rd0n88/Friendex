## Critical context: Phase 4 is UNAFFECTED by the guild dimension

Per ADR-0001 (Approach C), `guild_id` lives at the ORM/adapter layer and is
captured once via `container.for_guild(guild_id)`; **the domain layer stays
guild-agnostic.** Phase 4's pure functions take primitives and value objects —
`apply_trade_impact(current, shares, is_buy, k, min_price)`,
`compute_activity_return(bucket)`, etc. — never repository lookups. **Do NOT add
a `guild_id` parameter to any Phase 4 function.** This is affirmed explicitly in
`docs/06-per-guild-markets-migration.md` §3 ("Phase 4 — NO CHANGE").

## The one real interaction: the test conftest's `Settings` fixture

Phase 4 creates `tests/domain/conftest.py` with a `default_settings` fixture
(per the predecessor pass-baton's step 4). After PR #22, `Settings` no longer
accepts a required `guild_id`. **Branch Phase 4 from a `main` that already has
#22 merged**, and build the fixture as `Settings(discord_token="tmp", ...)` with
**no `guild_id`** (it is silently dropped by `extra="ignore"`, but leaving it is
stale/misleading). The pure functions themselves don't need `Settings` — they
take explicit constants — but the fixture may source those constants
(`price_impact_k`, `min_price`, `inactivity_decay`, …) from a `Settings`
instance.

## Open questions / risks

- **`datetime.utcnow()` deprecation** (flagged in the predecessor pass-baton):
  `market_hours.py` introduces clock-dependent logic. PR #13's 3.1 refinement
  may have addressed the model side — verify the domain uses timezone-aware UTC
  (`datetime.now(UTC)`) and that `conftest.py` pins time with `freezegun`.
- **Configurable game-rule toggles, not hardcoded constants:** several domain
  rules are "deferred to implementation phase" toggles in
  `docs/02-target-architecture.md` — APY period monthly vs annual (decision #8),
  the 50% VC-leave-boost condition (#6), hedge-fund single vs multi-investor
  (#5). `fund_math.py` / `market_hours.py` must honor the toggles.
- **`--cov` path form:** use the package-dir form `--cov=src/friendex/domain`
  (works — the Phase 3 gate used it). A single *module file* path triggers a
  "module never imported" warning; for one file use the module form
  (`--cov=friendex.domain.price_engine`).
