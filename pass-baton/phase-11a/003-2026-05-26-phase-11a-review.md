### LOW

- **LOW-1 — admin_cog docstring prose glitch.**
  `src/friendex/adapters/discord_bot/cogs/admin_cog.py:10-11`: the sentence
  "The command is gated by\n    with the *Manage Server* permission can
  broadcast the intro." appears truncated — a clause is missing (probably
  "such that only members" between "by" and "with"). The decorator is
  correctly applied; this is prose only. **Fix:** rephrase to "The command
  is gated by `@app_commands.checks.has_permissions(manage_guild=True)`, so
  only members with the *Manage Server* permission can broadcast the
  intro." (one-line edit, carry-forward to 11b/11c fix unit).

- **LOW-2 — `// type: ignore[no-untyped-def]` proliferation in cog tests.**
  Every test signature carries `# type: ignore[no-untyped-def]` to silence
  mypy on the pytest-fixture-typed parameters. This is the cogs-tests
  convention from earlier phases, but a clean fix would be to type the
  fixtures via a Protocol (e.g. `FakeInteractionFactory: Protocol`) so
  consumers can annotate explicitly. **Fix:** non-blocking; carry-forward
  if the 11b/11c slices want to invest. The current pattern works and
  matches Phase 8/9/10 test idioms.
